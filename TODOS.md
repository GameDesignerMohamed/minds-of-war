## Engineering — Code Fixes (from takeover review)

### High Priority

#### E1. Fix tick rate to 20Hz

**What:** Change `main.ts:615` from `tickRate: 1/60` to `tickRate: 1/20`, `maxFrameTime: 0.25`. One-line fix.
**Why:** Docs say 20Hz, code runs at 60Hz. 3x unnecessary CPU load. 20Hz is standard for RTS.

#### E2. Extract main.ts god file into 3-4 modules

**What:** Split 738-LOC main.ts into GameSetup.ts (renderer/scene/camera), GameCommands.ts (input/placement/harvest/train), GameRenderer.ts (render loop + HP bars), thin main.ts orchestrator.
**Why:** Every new feature adds to the god file. Biggest structural improvement.

#### E3. Define proper config interfaces, remove `as never` casts

**What:** Create typed interfaces for AIBehaviorProfile, FactionAbilityConfig, etc. Remove all 8 `as never` casts and 4 try-catch error-swallowing blocks in main.ts:185-213.
**Why:** Type mismatches are hidden. Config changes will silently break at runtime.

#### E4. Guard inside ResourceTracker.spend()

**What:** Add `if (!this.canAfford(playerId, cost)) return false;` at top of spend(). 5-line fix.
**Why:** Resources can go negative if caller forgets canAfford() check or two systems spend in same tick.

#### E5. Throw on missing unit defs in TrainingQueueSystem

**What:** Replace hardcoded fallback `trainTime: 20` with an error throw when unit def is missing.
**Why:** Silent fallback produces ghost units. Data bugs should fail loud.

#### E6. Write tests for 5 critical systems

**What:** AttackSystem, ConstructionSystem, TrainingQueueSystem, SpellSystem, victory condition. Currently zero tests for all of these.
**Why:** 45 tests exist but only for pure-function layer. Integration systems are untested.

#### E7. Write GameLoop + EventBus tests

**What:** Pure-logic classes with zero deps, trivially testable, zero tests today.
**Why:** GameLoop is the simulation heartbeat. EventBus handles all cross-system communication.

#### E10. Make BuffEntry.remainingDuration mutable

**What:** Remove readonly from mutated fields. Delete 3 `as unknown as` hacks in DisciplineAuraSystem.ts and SpellSystem.ts.

#### E11. Centralize player ID constants

**What:** Extract PLAYER_1 = 'human', PLAYER_2 = 'orc' constants. Replace all hardcoded string literals across codebase.
**Why:** 'human'/'orc' hardcoded everywhere. Makes future faction-select impossible without touching dozens of lines.

#### E12. Event-driven HP bars

**What:** Listen to UNIT_ATTACKED event to create/update HP bars instead of iterating all entities every frame at 60fps.
**Why:** Current approach iterates every entity with position every render frame. Wasted work for entities at full HP.

#### E13. ESLint + Prettier Setup

**Why:** No automated code quality enforcement. Inconsistencies grow with contributors.
